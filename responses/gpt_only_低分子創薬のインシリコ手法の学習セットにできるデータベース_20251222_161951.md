# アクセス方法の実務メモ（学習セット化のコツ）
1. **一括DL（FTP/Download）を優先**：再現性が高く、API制限にも引っかかりにくい（例：ChEMBL FTP、PubChem FTP、UniProt FTP）。  
2. **ID正規化が重要**：化合物（InChIKey/SMILES）、標的（UniProt accession）、疾患（MONDO/MeSH/DOID）を軸に突合。  
3. **ライセンス確認**：特にDrugBank、PDBbind、KEGG、商用カタログは要注意（モデル配布・データ再配布の可否）。  
4. **スプリット（リーク対策）**：同一骨格や類似化合物がtrain/testに跨ると性能が過大評価になりやすいので、scaffold split（Murcko）等を検討。

### GPT-mini + Thinking標準

- **モデル**: `gpt-5-mini`
- **reasoning.effort**: `medium`

以下は、低分子創薬（in silico）で機械学習用の学習データセットとして利用可能な主要データベースを網羅的に整理した一覧です。各データベースについて「概要」「代表的な利用目的（QSAR/仮想スクリーニング/ADMET/毒性/構造学習 等）」「アクセス方法（ダウンロード/API/フォーマット）」「利用上の注意（ライセンス・登録等）」を記載します。最後に、実務上の前処理・統合・ベンチマークの注意点と推奨ツールも簡潔にまとめます。

1) 化合物構造・化学空間（大規模ライブラリ）
- PubChem
  - 概要：数億件の化学物質とPubChem BioAssayによる活性データを含む最大級のリポジトリ。
  - 用途：汎用の構造データ、HTSバイオアッセイ、陽性/陰性ラベル取得。
  - アクセス：ウェブUI、FTP（ftp.ncbi.nlm.nih.gov/pub/chem...）、PUG-REST API（https://pubchem.ncbi.nlm.nih.gov/）／フォーマット：SDF, CSV, JSON。
  - 注意：二次情報（混同する同一体）や標準化が必要。

- ZINC (ZINC15 / ZINC20)
  - 概要：購入可能な化合物集合やドッキング用のプリコンピュートされたファイル群。
  - 用途：バーチャルライブラリ、生成モデルの学習、デノボ・探索空間。
  - アクセス：ウェブ（https://zinc15.docking.org/ や https://zinc20.docking.org/）、一括ダウンロード可能（SMILES/SDF）。
  - 注意：商用利用規約やベンダーごとの制約に注意。

- GDBシリーズ（GDB-17 等）
  - 概要：化学構造の列挙データベース（理論上の化合物空間）。
  - 用途：生成モデルの学習、化学空間分析。
  - アクセス：プロジェクトサイトやFigshare等でダウンロード（SMILES等）。

- eMolecules / MolPort / Enamine / commercial libraries
  - 概要：市販化合物カタログ（購入可）。
  - 用途：スクリーニングライブラリ、リアリスティックな候補探索。
  - アクセス：アカウント登録後に一括ダウンロード/検索API（ベンダー毎）。
  - 注意：ライセンス・商用条件が強い。

2) バイオアクティビティ（バインディング、IC50, Ki 等）
- ChEMBL
  - 概要：論文・特許由来のキュレーション済み生物活性（pIC50等）と標的アノテーション。
  - 用途：QSAR、標的指向学習、ADMET関連解析。
  - アクセス：ウェブUI、FTP（https://ftp.ebi.ac.uk/pub/databases/chembl/）、REST API（https://www.ebi.ac.uk/chembl/ws）／フォーマット：TSV, SQLite, SDF。
  - 注意：同一化合物の重複・単位の統一が必要。

- BindingDB
  - 概要：結合親和性（Ki, Kd, IC50等）のデータベース（タンパク−リガンド）。
  - 用途：結合予測の教師データ、pKd/pIC50回帰。
  - アクセス：ウェブ、ダウンロードページ（https://www.bindingdb.org/ から各種TSV/CSV）、APIも一部あり。
  - 注意：測定条件のメタデータ確認が重要。

- PubChem BioAssay（PubChem内）
  - 概要：多数のHTSアッセイ結果（アクティブ/インアクティブ等）。
  - 用途：分類タスク（活性/非活性）、機械学習の陽性率学習。
  - アクセス：PubChem GUI/API/FTP。

3) タンパク質構造・複合体（3D 構造データ）
- PDB (RCSB PDB)
  - 概要：タンパク質・核酸の実験構造（X-ray, NMR）と複合体。
  - 用途：ドッキング/構造ベース学習、3Dグリッド・グラフ学習。
  - アクセス：ウェブ、FTP（ftp.wwpdb.org/pub/pdb/）およびREST API（https://data.rcsb.org/）／フォーマット：PDB/mmCIF。
  - 注意：結合ポーズの整合性、解像度情報の確認。

- PDBbind
  - 概要：PDB複合体に対して結合親和性情報を付与したセット（トレーニング用に広く使われる）。
  - 用途：結合自由エネルギー回帰、スコアリング関数学習。
  - アクセス：専用サイト（http://www.pdbbind.org.cn/）から登録してダウンロード（データセット毎に配布）。
  - 注意：利用時にライセンス/引用の明記が必要。

- AlphaFold DB
  - 概要：タンパク質構造の高精度予測（DeepMind/EMBL-EBI）。
  - 用途：標的構造の補完、構造ベースの特徴生成。
  - アクセス：ウェブおよびFTP（https://alphafold.ebi.ac.uk/）。

4) ADMET / 毒性関連データ
- ToxCast / Tox21（EPA / NCATS）
  - 概要：多数の in vitro 毒性アッセイデータ。Tox21は共同プロジェクト。
  - 用途：毒性予測モデル、プロファイリング。
  - アクセス：EPAサイト／FTP、データはCSV形式で配布。PubChemにも一部登録。
  - 注意：アッセイごとの条件差や閾値の扱いが必要。

- DSSTox（EPA）
  - 概要：化学物質の毒性情報データベース。ID体系の整備が進む。
  - アクセス：EPA DSSToxサイト、ダウンロード可能。

- SIDER
  - 概要：医薬品の副作用データ（製品ラベル由来）。
  - 用途：臨床毒性/副作用予測。
  - アクセス：http://sideeffects.embl.de/ からダウンロード（TSV）。

- ClinTox / OFFSIDES / FAERS / OpenFDA
  - 概要：臨床副作用・有害事象報告（FAERSはFDAの報告データ）。
  - アクセス：FAERSはFDAサイトおよびOpenFDA API（https://open.fda.gov/）。ClinToxはMoleculeNetに含まれることがある。

5) 薬剤情報・標的・臨床データ
- DrugBank
  - 概要：薬剤の詳細（作用機序、標的、薬物性状、臨床情報）。
  - 用途：既存薬の類推学習、薬物再配置。
  - アクセス：ウェブ（https://go.drugbank.com/）。完全版はアカウント登録・利用条件あり（XML/CSV）。
  - 注意：商用利用時のライセンスを確認。

- DrugCentral
  - 概要：薬剤データベース（薬効、標的、薬物相互作用）。
  - アクセス：ウェブおよび一括ダウンロード（TSV等）。

6) ベンチマーク／既成MLデータセット
- MoleculeNet
  - 概要：QSARや物性・毒性予測の代表的ベンチマーク群（ESOL, FreeSolv, Lipophilicity, BACE, BBBP, Tox21, ToxCast, SIDER, ClinTox 等）。
  - 用途：MLアルゴリズムの比較・評価。
  - アクセス：http://moleculenet.ai/、DeepChemライブラリ経由で簡単に取得（Python）。
  - 注意：各データセットの分割方法（random/scaffold/time）に注意。

- QM9
  - 概要：小分子の量子化学的性質（DFT計算結果）。

利用上の注意（ライセンス・倫理・品質）
- ライセンス：DrugBankや商用ライブラリ、特定の企業データは商用利用に制限あり。必ず利用規約を確認してから使用してください。
- データ品質：実験条件（バッファ、温度、測定法）が異なると数値が混ざりやすい。IC50/Ki 等は単位・測定系の正規化（pIC50化）を行う。
- 重複と標準化：同一化合物が別IDで登録されていることが多い（塩、溶媒、異性体、タウトマー）。SMILES正規化、塩除去、標準化（RDKit, MolVS, ChEMBL Structure Pipeline 推奨）を行う。

前処理・統合に有用なツール・ベストプラクティス

- 構造化ツール：Biopython、MDAnalysis（タンパク質処理）、PyMOL/MDtraj（可視化）。
- ML向けライブラリ：DeepChem、TorchDrug、DGL-LifeSci、Open Graph Benchmark（化学向け実装）。
- 前処理手順（推奨）
  1. IDマッピング：InChIKeyをキーに照合し重複検出。
  2. 塩・溶媒除去、プロトン化状態の統一（pHでの標準化）。
  3. 単位・値の正規化（IC50→pIC50、log-transform）。
  4. アッセイ品質フィルタ（繰り返し数、測定法、出典を考慮）。
  5. データ分割：scaffold-split を推奨（化学空間リーケージを防ぐ）。
  6. 欠損値・検出限界の扱い（Censored data: >,< を扱う手法を適用）。

タスク別オススメデータセット（すぐ使える）

- バイナリ活性分類（HTS）：PubChem BioAssay, ChEMBL bioassays

- 量子化学予測：QM9

最後に：推奨ワークフローの簡潔な提案
- 目的（QSAR, ドッキングスコア学習, 毒性予測等）を定め、該当データベース（ChEMBL/BindingDB/Tox21等）を選定。

# 学習セット化で必ず問題になる点（最小限チェックリスト）

# アクセス方法の実務メモ（学習セット化のコツ）
1. **一括DL（FTP/Download）を優先**：再現性が高く、API制限にも引っかかりにくい（例：ChEMBL FTP、PubChem FTP、UniProt FTP）。  
2. **ID正規化が重要**：化合物（InChIKey/SMILES）、標的（UniProt accession）、疾患（MONDO/MeSH/DOID）を軸に突合。  
3. **ライセンス確認**：特にDrugBank、PDBbind、KEGG、商用カタログは要注意（モデル配布・データ再配布の可否）。  
4. **スプリット（リーク対策）**：同一骨格や類似化合物がtrain/testに跨ると性能が過大評価になりやすいので、scaffold split（Murcko）等を検討。
