### GPT-mini + Thinking標準

- **モデル**: `gpt-5-mini`
- **reasoning.effort**: `medium`

要点：L1000（約900のランドマーク遺伝子）以外に、ノックダウン（KD）、オーバーエクスプレッション（OE）、薬剤（化合物）摂動について「全遺伝子（全転写子）発現」を取得できる主要なデータソースを網羅的にまとめます。各リソースの特徴、入手方法（検索／ダウンロード手順）、注意点も併せて示します。用途（bulk RNA‑seq / マイクロアレイ / scRNA‑seq / 推定フルトランスクリプトーム）によって推奨先が変わるので、その点も明記します。

1) NCBI Gene Expression Omnibus (GEO)
- 内容：マイクロアレイ／RNA‑seqを中心に、KD/OE/薬剤摂動の多数の公開実験（Series: GSE、Samples: GSM）。オリジナルの薬剤摂動マイクロアレイ実験（旧Connectivity Map以前の研究等）や個別のKD/OE論文データが豊富。
- なぜ全遺伝子が取れるか：マイクロアレイはプローブで全遺伝子近似、RNA‑seqはリードから遺伝子カウントを作成可。
- 入手方法：
  - GEOウェブ（https://www.ncbi.nlm.nih.gov/geo/）で "knockdown", "overexpression", "drug", "treatment" などキーワード検索。フィルタで organism/assay type を絞る。
  - rawデータ（FASTQ/CEL等）はSRAまたはGEO FTPで取得。GSEのページからSupplementaryやSRAへのリンクがある。
  - コマンド例：
    - GEO FTP: ftp://ftp.ncbi.nlm.nih.gov/geo/series/GSEnnn/GSEXXXXX/（適切な系列ディレクトリを参照）
    - SRAから：prefetch SRRxxxx; fasterq-dump SRRxxxx（sra-toolkit）
- 注意点：メタデータの記載ゆれが多い（"knockdown" が "KD" と書かれている等）。自動抽出はメタデータクリーニングが必要。

2) ArrayExpress / Expression Atlas（EMBL‑EBI）
- 内容：マイクロアレイ／RNA‑seq実験データ。Expression Atlasは処理済み差次的発現（デフォルトで "baseline" / "differential"）を提供し、"contrasts" タグで薬剤や遺伝子摂動の比較が検索しやすい。
- 入手方法：
  - ArrayExpress（https://www.ebi.ac.uk/arrayexpress/）でキーワード検索（"knockdown","overexpression","drug"）。
  - Expression Atlas（https://www.ebi.ac.uk/gxa/）の "Experiment search" で "perturbation" や "knockdown" を検索。processed matrix や差分結果をダウンロード可（FTP/APIあり）。
  - FTP/API使用でバルク取得可能。
- 注意点：Count/TPM など処理形式が揃っている実験とそうでないものが混在。

3) SRA / ENA（生のシーケンスリード／メタデータ）
- 内容：RNA‑seqの生データ（FASTQ）。GEOにリンクがあるものは同じくSRAに登録されていることが多い。ENA（EMBL‑EBI）も同様。
- 入手方法：
  - SRA Run Browser（https://www.ncbi.nlm.nih.gov/sra）でキーワード検索。
  - sra‑toolkit（prefetch、fasterq‑dump）やENA FTP/Asperaでダウンロード。
- 注意点：生データから一貫したパイプライン（STAR/Kallisto/Salmon → featureCounts/tximport）で再処理して利用することを推奨。

4) ENCODE（ENCyclopedia Of DNA Elements）
- 内容：ヒト・マウスを中心に、CRISPRやRNAiによる摂動、化合物処理のRNA‑seq実験など質の高い摂動データが多数ある。メタデータが詳細。
- 入手方法：
  - ENCODE portal（https://www.encodeproject.org/）で search → assays: "RNA-seq"、perturbation_type: "knockdown/knockout/drug treatment" 等で絞り込み。
  - 各実験のprocessedファイル（gene counts / bigWig）を直接ダウンロード可能。
- 注意点：ヒト・モデル細胞系が中心で、標準化された処理済みファイルを使える利点が大きい。

5) Connectivity Map / LINCS 関連（Broad Institute）
- 内容：代表的なLarge‑scale薬剤/遺伝子摂動コンパニオン。主にL1000（978ランドマーク）だが、LINCSプロジェクトやBroadに関連するサブセットでRNA‑seq（フル）データや「ランドマークから推定した全遺伝子推定値（inferred）」を提供している場合がある。
- 入手方法：
  - LINCS Data Portal / CLUE（https://clue.io/）でデータ検索。download optionsに "inferred" や一部のRNA‑seqサブセットがある。
  - iLINCS（http://ilincs.org/）やL1000FWDなどのツール経由で推定全遺伝子シグネチャを利用可能（ただし「推定値」である点に注意）。
- 注意点：L1000の「inferred」全遺伝子はモデル推定であり、実測RNA‑seqとは異なる。解析目的によっては適切でない場合あり。

6) CREEDS（Crowd Extracted Expression of Differential Signatures）
- 内容：文献/GEOから抽出した「遺伝子摂動（KD/OE）」「薬剤/疾患」ごとの差次的発現シグネチャ（フル遺伝子リストのup/down）。研究目的で使いやすく整理されている。
- 入手方法：
  - CREEDSサイト/データ配布（https://maayanlab.cloud/CREEDS/）やGitHubからダウンロード。API／TSV形式あり。
- 注意点：元データはGEO等に依存。シグネチャはカーブ化済み（要検証・元データ参照を推奨）。

7) ARCHS4（All RNA‑seq Samples from GEO/SRA, processed）
- 内容：GEO/SRAの大量RNA‑seqデータを統一処理して遺伝子レベルカウント/TPMを提供。メタデータ検索で「perturbation」系のサンプルを抽出できる。
- 入手方法：
  - ARCHS4ウェブ（https://maayanlab.cloud/archs4/）またはAPIでダウンロード（large HDF5/TSVファイル提供）。
- 注意点：メタデータの完全性に依存。摂動種別の抽出は手作業でのフィルタリングが必要。

8) recount / Recount3
- 内容：SRA由来のRNA‑seqを統一処理した大規模コーパス。プロジェクトごとのメタデータがあるため、KD/OE/薬剤実験を抽出して利用可能。
- 入手方法：
  - recountウェブ/ Bioconductorパッケージ（recount, recount3）でダウンロード。Rから簡単に取得可。
- 注意点：解析に便利な一貫処理済みカウントを提供。メタデータのクリーニングは必要。

9) scRNA‑seqによる摂動データ（Perturb‑seq, CROP‑seq 等）
- 内容：CRISPRi/a, CRISPR knockout, ORF overexpression 等を単一細胞RNA‑seqで測定したデータ群。個々のガイドRNAやoverexpression対象ごとの全転写子応答が得られる（生データはフル転写子）。
- 代表例/入手先：
  - 主要論文のGEO/SRA（例: Adamson et al., Dixit et al., Replogle 等）。具体的なGSEは論文ページ参照。
  - scPerturb.org やSingle Cell Portal/ArrayExpressにもコレクションあり（scPerturb系のまとめサイトを検索）。
- 入手方法：GEO/SRA経由でraw/processedデータを取得後、専用パイプラインでガイド→発現対応付け（CROPseqTools, Scanpy, Seurat+相关ツール）。
- 注意点：分析が高度（ガイド割当・ドロップアウト補正・バッチ補正等）。だが標的ごとの全遺伝子応答解析には最適。

10) 毒性／薬理学特化データベース：TG‑GATEs、DrugMatrix
- 内容：大量の化合物投与（in vitro / in vivo）のマイクロアレイ／RNA‑seqデータ。毒性評価や薬物応答研究に使える全遺伝子発現データ。
- 入手方法：
  - Open TG‑GATEs（https://toxico.nibiohn.go.jp/）でダウンロード。DrugMatrixはNTP/公共リポジトリやArrayExpress経由で入手可能（利用条件あり）。
- 注意点：動物（ラット）由来データが多い点やドーズ/時系列情報が豊富。

11) その他の統合ツール／データ集約サービス
- Enrichr（およびEnrichrの "Drug Perturbations from GEO", "Gene Perturbations from Enrichr" ライブラリ）
- iLINCS：LINCS系の解析とデータベース（推定全遺伝子や処理済みsignature）。
- Harmonizome（遺伝子属性の統合）
- GEOmetadb（Rパッケージ）でGEOメタデータをバルク検索

実践的検索／取得ワークフロー（簡潔）
1. 目的を決める：対象生物種（ヒト/マウス/ラット）、アッセイ（bulk RNA‑seq / microarray / scRNA‑seq）、摂動タイプ（RNAi/siRNA/CRISPR/KD/OE/compound）、実験系（細胞株/一次細胞/組織）。
2. 大規模リポジトリでキーワード検索：
   - GEO/ArrayExpress/Expression Atlas："knockdown", "knock‑down", "siRNA", "shRNA", "CRISPRi", "overexpression", "ORF", "drug", "compound", "treatment" を組み合わせ。
3. 生データが欲しければSRA/ENAへ（sra‑toolkit, fasterq‑dump を使用）。
4. 大量バッチで処理したい場合：recount/ARCHS4/ExpressionAtlasの処理済みマトリクスを先に検討。
5. scRNA摂動は論文ごとに手作業で抽出することが多い（scPerturbまとめを参照）。
6. 一貫解析：同一パイプライン（アライメント→カウント→正規化→差分）で再処理することを推奨。

重要な注意点
- L1000の「inferred」全遺伝子は便利だが観測値ではない。検証やダウンストリーム解析の解釈に注意。
- メタデータの不統一（"knockdown" の表記ゆれ等）により、自動抽出は誤検出や見落としが起きやすい。
- マイクロアレイ vs RNA‑seq でバイアスあり。可能なら同一技術内で統一して比較を。
- ライセンス・再利用条件を確認（特に企業利用時）。

もし希望があれば（対応可能内容）
- 特定の生物種（ヒト/マウス/ラット）、実験様式（bulk RNA‑seq / scRNA‑seq / マイクロアレイ）の組み合わせを教えていただければ、該当する代表的なGSEアクセッション一覧（実際にKD/OE/薬剤のフル遺伝子発現が得られるもの）を抽出してリスト化し、ダウンロードコマンド付きで提供します。
- あるいは「薬剤摂動でヒト肝細胞、RNA‑seq、過去10年分」など条件を指定いただければ、それに合うデータセットを具体的に列挙します。

必要であれば、まず「対象（ヒトかマウスか）」「bulkかsingle‑cellか」「KD/OE/薬剤どれを優先か」等を教えてください。そうすれば具体的なアクセッションとダウンロード手順（コマンド例つき）を用意します。

## 1) LINCSで「全遺伝子」を使う（最優先候補）
### 1-1. LINCS “RNA-seq”（KD/OE/化合物を含む、ただしL1000ほど網羅ではない）

- **摂動タイプ**：化合物、shRNA/siRNA、CRISPR、cDNA過剰発現（プロジェクト／フェーズで範囲が変動）
- **測定**：RNA-seq（全遺伝子）、または一部はマイクロアレイ
- **入手先（代表）**
  - **iLINCS（Interactive LINCS）**：検索・可視化・ダウンロードがしやすい

**ポイント**：  
- 「L1000は約900遺伝子しか実測がない」問題に対して、LINCS内でも**RNA-seq実測**を選べば全遺伝子が取れます。

## 4) KD/OE/薬剤の“全遺伝子”を一番広く集める：GEO / SRA / ArrayExpress（網羅性最大）
「特定の巨大プロジェクトだけ」だとKD/OEが不足しがちなので、**公共リポジトリ横断**が結局もっとも網羅的です。

### 4-1. GEO（Gene Expression Omnibus）
- **強み**：KD/OE（siRNA/shRNA/CRISPR/KO/OE）も薬剤処理も、とにかく件数が多い。  
- **データ形態**：RNA-seq（counts/FASTQはSRA側が多い）、マイクロアレイ（CEL含む）
- **アクセス**：https://www.ncbi.nlm.nih.gov/geo/
- **入手方法（代表）**
  1) GEOでキーワード検索  
     - KD系：`shRNA`, `siRNA`, `knockdown`, `CRISPR`, `knockout`, `KO`  
     - OE系：`overexpression`, `cDNA`, `inducible`, `Tet-On`  
     - 薬剤：化合物名 + `treatment`, `compound`, `inhibitor`  
     - 例：`("RNA-seq") AND (CRISPR OR knockdown OR overexpression) AND (cell line)`  
  2) シリーズ（GSE）から**processed data**をダウンロード  
  3) 生RNA-seqが必要なら、GSE→SRAへのリンクからFASTQ取得（次項）

- **プログラム取得**
  - R: `GEOquery` パッケージでGSEを取得（processed中心）
  - 生データはSRA Toolkit（下記）

### 4-2. SRA（Sequence Read Archive）— 生RNA-seq FASTQを確実に取る
- **アクセス**：https://www.ncbi.nlm.nih.gov/sra
- **入手方法**
  - SRA Toolkit（`prefetch`, `fasterq-dump`）でFASTQ取得  
- **利点**：全遺伝子の再定量（同一パイプラインで再解析）が可能 → 異なる研究間の比較がやりやすい

### 4-3. ArrayExpress / Expression Atlas（EMBL-EBI）
- **内容**：欧州側の受け皿。GEOと重複もあるが、メタデータが使いやすい場合あり。
- **アクセス**：
  - ArrayExpress: https://www.ebi.ac.uk/arrayexpress/
  - Expression Atlas: https://www.ebi.ac.uk/gxa/home
- **入手方法**：実験（E-MTABなど）単位でダウンロード、もしくはFTP/API経由

## 追加で確認したい点（これが分かると「網羅的調査」をあなたの用途に最適化できます）
1) 対象は **ヒト細胞株のみ**ですか？（マウス/ラットのin vivoも含めますか：TG-GATEs等が効いてきます）  
2) 必要なのは **バルクRNA-seqのみ**ですか？単一細胞（Perturb-seq）も許容ですか？  
3) KDは **shRNA/siRNA**限定ですか、それとも **CRISPR KO/CRISPRi**もOKですか？  
4) 「全遺伝子」は **実測（RNA-seq）必須**か、**全ゲノムマイクロアレイ**も可ですか？

上の条件を教えてくれれば、(i) LINCS内で実際に「RNA-seqでKD/OE/薬剤が取れる」範囲の当たり付け、(ii) GEO/ENCODEでの具体的な検索式、(iii) 取得スクリプト（SRA Toolkit/R/Python）まで含めて、より実務的な“収集リスト”に落とし込みます。

## 0. 前提：L1000でも「推定全遺伝子」があるが、“実測”ではない
LINCS L1000はご指摘の通りランドマーク遺伝子（約978）を実測ですが、データとしては多くの場合、**推定（inferred）で数千〜1万超の遺伝子を補完した行列**も配布されています。  
ただし「推定は使いにくい／実測が必要」という要件なら、以下の **RNA-seq または（全遺伝子測定の）マイクロアレイ**系に移るのが基本方針になります。

## 統合後の出力

要点：L1000（約900のランドマーク遺伝子）以外に、ノックダウン（KD）、オーバーエクスプレッション（OE）、薬剤（化合物）摂動について「全遺伝子（全転写子）発現」を取得できる主要なデータソースを網羅的にまとめます。各リソースの特徴、入手方法（検索／ダウンロード手順）、注意点も併せて示します。用途（bulk RNA‑seq / マイクロアレイ / scRNA‑seq / 推定フルトランスクリプトーム）によって推奨先が変わるので、その点も明記します。

1) NCBI Gene Expression Omnibus (GEO)
- 内容：マイクロアレイ／RNA‑seqを中心に、KD/OE/薬剤摂動の多数の公開実験（Series: GSE、Samples: GSM）。オリジナルの薬剤摂動マイクロアレイ実験（旧Connectivity Map以前の研究等）や個別のKD/OE論文データが豊富。
- なぜ全遺伝子が取れるか：マイクロアレイはプローブで全遺伝子近似、RNA‑seqはリードから遺伝子カウントを作成可。
- 入手方法：
  - GEOウェブ（https://www.ncbi.nlm.nih.gov/geo/）で "knockdown", "overexpression", "drug", "treatment" などキーワード検索。フィルタで organism/assay type を絞る。
  - rawデータ（FASTQ/CEL等）はSRAまたはGEO FTPで取得。GSEのページからSupplementaryやSRAへのリンクがある。
  - コマンド例：
    - GEO FTP: ftp://ftp.ncbi.nlm.nih.gov/geo/series/GSEnnn/GSEXXXXX/（適切な系列ディレクトリを参照）
    - SRAから：prefetch SRRxxxx; fasterq-dump SRRxxxx（sra-toolkit）
- 注意点：メタデータの記載ゆれが多い（"knockdown" が "KD" と書かれている等）。自動抽出はメタデータクリーニングが必要。

2) ArrayExpress / Expression Atlas（EMBL‑EBI）
- 内容：マイクロアレイ／RNA‑seq実験データ。Expression Atlasは処理済み差次的発現（デフォルトで "baseline" / "differential"）を提供し、"contrasts" タグで薬剤や遺伝子摂動の比較が検索しやすい。
- 入手方法：
  - ArrayExpress（https://www.ebi.ac.uk/arrayexpress/）でキーワード検索（"knockdown","overexpression","drug"）。
  - Expression Atlas（https://www.ebi.ac.uk/gxa/）の "Experiment search" で "perturbation" や "knockdown" を検索。processed matrix や差分結果をダウンロード可（FTP/APIあり）。
  - FTP/API使用でバルク取得可能。
- 注意点：Count/TPM など処理形式が揃っている実験とそうでないものが混在。

3) SRA / ENA（生のシーケンスリード／メタデータ）
- 内容：RNA‑seqの生データ（FASTQ）。GEOにリンクがあるものは同じくSRAに登録されていることが多い。ENA（EMBL‑EBI）も同様。
- 入手方法：
  - SRA Run Browser（https://www.ncbi.nlm.nih.gov/sra）でキーワード検索。
  - sra‑toolkit（prefetch、fasterq‑dump）やENA FTP/Asperaでダウンロード。
- 注意点：生データから一貫したパイプライン（STAR/Kallisto/Salmon → featureCounts/tximport）で再処理して利用することを推奨。

4) ENCODE（ENCyclopedia Of DNA Elements）
- 内容：ヒト・マウスを中心に、CRISPRやRNAiによる摂動、化合物処理のRNA‑seq実験など質の高い摂動データが多数ある。メタデータが詳細。
- 入手方法：
  - ENCODE portal（https://www.encodeproject.org/）で search → assays: "RNA-seq"、perturbation_type: "knockdown/knockout/drug treatment" 等で絞り込み。
  - 各実験のprocessedファイル（gene counts / bigWig）を直接ダウンロード可能。
- 注意点：ヒト・モデル細胞系が中心で、標準化された処理済みファイルを使える利点が大きい。

5) Connectivity Map / LINCS 関連（Broad Institute）
- 内容：代表的なLarge‑scale薬剤/遺伝子摂動コンパニオン。主にL1000（978ランドマーク）だが、LINCSプロジェクトやBroadに関連するサブセットでRNA‑seq（フル）データや「ランドマークから推定した全遺伝子推定値（inferred）」を提供している場合がある。
- 入手方法：
  - LINCS Data Portal / CLUE（https://clue.io/）でデータ検索。download optionsに "inferred" や一部のRNA‑seqサブセットがある。
  - iLINCS（http://ilincs.org/）やL1000FWDなどのツール経由で推定全遺伝子シグネチャを利用可能（ただし「推定値」である点に注意）。
- 注意点：L1000の「inferred」全遺伝子はモデル推定であり、実測RNA‑seqとは異なる。解析目的によっては適切でない場合あり。

6) CREEDS（Crowd Extracted Expression of Differential Signatures）
- 内容：文献/GEOから抽出した「遺伝子摂動（KD/OE）」「薬剤/疾患」ごとの差次的発現シグネチャ（フル遺伝子リストのup/down）。研究目的で使いやすく整理されている。
- 入手方法：
  - CREEDSサイト/データ配布（https://maayanlab.cloud/CREEDS/）やGitHubからダウンロード。API／TSV形式あり。
- 注意点：元データはGEO等に依存。シグネチャはカーブ化済み（要検証・元データ参照を推奨）。

7) ARCHS4（All RNA‑seq Samples from GEO/SRA, processed）
- 内容：GEO/SRAの大量RNA‑seqデータを統一処理して遺伝子レベルカウント/TPMを提供。メタデータ検索で「perturbation」系のサンプルを抽出できる。
- 入手方法：
  - ARCHS4ウェブ（https://maayanlab.cloud/archs4/）またはAPIでダウンロード（large HDF5/TSVファイル提供）。
- 注意点：メタデータの完全性に依存。摂動種別の抽出は手作業でのフィルタリングが必要。

8) recount / Recount3
- 内容：SRA由来のRNA‑seqを統一処理した大規模コーパス。プロジェクトごとのメタデータがあるため、KD/OE/薬剤実験を抽出して利用可能。
- 入手方法：
  - recountウェブ/ Bioconductorパッケージ（recount, recount3）でダウンロード。Rから簡単に取得可。
- 注意点：解析に便利な一貫処理済みカウントを提供。メタデータのクリーニングは必要。

9) scRNA‑seqによる摂動データ（Perturb‑seq, CROP‑seq 等）
- 内容：CRISPRi/a, CRISPR knockout, ORF overexpression 等を単一細胞RNA‑seqで測定したデータ群。個々のガイドRNAやoverexpression対象ごとの全転写子応答が得られる（生データはフル転写子）。
- 代表例/入手先：
  - 主要論文のGEO/SRA（例: Adamson et al., Dixit et al., Replogle 等）。具体的なGSEは論文ページ参照。
  - scPerturb.org やSingle Cell Portal/ArrayExpressにもコレクションあり（scPerturb系のまとめサイトを検索）。
- 入手方法：GEO/SRA経由でraw/processedデータを取得後、専用パイプラインでガイド→発現対応付け（CROPseqTools, Scanpy, Seurat+相关ツール）。
- 注意点：分析が高度（ガイド割当・ドロップアウト補正・バッチ補正等）。だが標的ごとの全遺伝子応答解析には最適。

10) 毒性／薬理学特化データベース：TG‑GATEs、DrugMatrix
- 内容：大量の化合物投与（in vitro / in vivo）のマイクロアレイ／RNA‑seqデータ。毒性評価や薬物応答研究に使える全遺伝子発現データ。
- 入手方法：
  - Open TG‑GATEs（https://toxico.nibiohn.go.jp/）でダウンロード。DrugMatrixはNTP/公共リポジトリやArrayExpress経由で入手可能（利用条件あり）。
- 注意点：動物（ラット）由来データが多い点やドーズ/時系列情報が豊富。

11) その他の統合ツール／データ集約サービス
- Enrichr（およびEnrichrの "Drug Perturbations from GEO", "Gene Perturbations from Enrichr" ライブラリ）
- iLINCS：LINCS系の解析とデータベース（推定全遺伝子や処理済みsignature）。
- Harmonizome（遺伝子属性の統合）
- GEOmetadb（Rパッケージ）でGEOメタデータをバルク検索

実践的検索／取得ワークフロー（簡潔）
1. 目的を決める：対象生物種（ヒト/マウス/ラット）、アッセイ（bulk RNA‑seq / microarray / scRNA‑seq）、摂動タイプ（RNAi/siRNA/CRISPR/KD/OE/compound）、実験系（細胞株/一次細胞/組織）。
2. 大規模リポジトリでキーワード検索：
   - GEO/ArrayExpress/Expression Atlas："knockdown", "knock‑down", "siRNA", "shRNA", "CRISPRi", "overexpression", "ORF", "drug", "compound", "treatment" を組み合わせ。
3. 生データが欲しければSRA/ENAへ（sra‑toolkit, fasterq‑dump を使用）。
4. 大量バッチで処理したい場合：recount/ARCHS4/ExpressionAtlasの処理済みマトリクスを先に検討。
5. scRNA摂動は論文ごとに手作業で抽出することが多い（scPerturbまとめを参照）。
6. 一貫解析：同一パイプライン（アライメント→カウント→正規化→差分）で再処理することを推奨。

重要な注意点
- L1000の「inferred」全遺伝子は便利だが観測値ではない。検証やダウンストリーム解析の解釈に注意。
- メタデータの不統一（"knockdown" の表記ゆれ等）により、自動抽出は誤検出や見落としが起きやすい。
- マイクロアレイ vs RNA‑seq でバイアスあり。可能なら同一技術内で統一して比較を。
- ライセンス・再利用条件を確認（特に企業利用時）。

もし希望があれば（対応可能内容）
- 特定の生物種（ヒト/マウス/ラット）、実験様式（bulk RNA‑seq / scRNA‑seq / マイクロアレイ）の組み合わせを教えていただければ、該当する代表的なGSEアクセッション一覧（実際にKD/OE/薬剤のフル遺伝子発現が得られるもの）を抽出してリスト化し、ダウンロードコマンド付きで提供します。
- あるいは「薬剤摂動でヒト肝細胞、RNA‑seq、過去10年分」など条件を指定いただければ、それに合うデータセットを具体的に列挙します。

必要であれば、まず「対象（ヒトかマウスか）」「bulkかsingle‑cellか」「KD/OE/薬剤どれを優先か」等を教えてください。そうすれば具体的なアクセッションとダウンロード手順（コマンド例つき）を用意します。
